¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de redes interconectadas que utiliza el protocolo TCP/IP para transmitir datos. La World Wide Web (WWW) es un sistema de información basado en hipertexto que opera a través de Internet y utiliza el protocolo HTTP para transferir datos.

¿Cuáles son las partes de un URL?

Las partes de un URL son: protocolo (http, https, etc.), dominio (www.ejemplo.com), ruta (carpetas y archivos), parámetros (?param1=valor1, param2=valor2) y ancla (#seccion).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicitar un recurso.
HEAD: Solicitar los encabezados de un recurso.
POST: Enviar datos para crear un nuevo recurso.
PUT: Actualizar un recurso existente.
PATCH: Actualizar parcialmente un recurso existente.
DELETE: Eliminar un recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar el método POST, ya que los datos enviados (usuario y contraseña) no deben formar parte de la URL visible y accesible en el historial del navegador.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET, ya que se está solicitando un recurso (la página web) al servidor.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 significa "OK" o "Exitoso". No ocurrió ningún error, la solicitud se procesó correctamente.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir el error 404 (Recurso no encontrado), ya que esto significa que el usuario está intentando acceder a una URL que no existe en el servidor web.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir el error 500 (Error interno del servidor), ya que esto indica un problema en el código o la configuración del servidor web.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Que un atributo esté depreciado o desaprobado significa que ya no se recomienda su uso y probablemente será eliminado en futuras versiones del estándar. Algunos elementos desaprobados de HTML 4 en HTML5 son: <center>, <font>, <strike>, <frame>, <frameset>, entre otros.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Algunas diferencias principales son: nuevos elementos semánticos en HTML5 (<header>, <nav>, <article>, etc.), mejor soporte para multimedia (audio y video), nuevos tipos de entrada en formularios, soporte para aplicaciones web offline, entre otras.

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla tiene los siguientes componentes:

  • Estructura: <table>, <tr>, <th>, <td>
  • Estilos: border, width, height, text-align, vertical-align, entre otros.

¿Cuáles son los principales controles de una forma HTML5?

Algunos de los principales controles de una forma HTML5 son:

  • <input> (text, email, password, checkbox, radio, etc.)
  • <textarea>
  • <select> y <option>
  • <button>
  • Nuevos tipos de entrada como color, date, range, etc.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información es un enfoque por fases para el análisis y diseño que comprende las siguientes etapas: investigación preliminar, determinación de requerimientos, diseño, desarrollo, integración y prueba, implementación y mantenimiento.

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información es un proceso iterativo e incremental que comprende las siguientes fases: planificación, análisis, diseño, implementación y mantenimiento.
